# 修复少通道实验脚本

## 问题分析

通过查看日志文件，我发现了以下问题：

1. **通道选择未生效**：所有网络都使用了默认通道列表，没有使用指定的通道配置（FP1 FP2 T7 T8 或 FP1 FP2 T7 T8 F7 F8 CP5 CP6）

2. **GPU设备配置错误**：

   * ACRNN遇到CUDA内存不足错误

   * EEGNet遇到无效设备序号错误

3. **结果提取逻辑错误**：脚本假设输出中包含 "acc: xxx" 和 "macro-f1: xxx" 格式，但实际输出格式不同

4. **命令行输出问题**：训练输出被重定向到日志文件，命令行看不到实时输出

## 修复方案

### 1. 修复通道选择

* 确保通道参数正确传递给每个网络

* 修复部分网络的通道选择逻辑（如R2GSTNN和RGNN）

* 确保通道名称正确转换为索引

### 2. 修复GPU设备配置

* 统一使用可用的GPU设备

* 为不同网络分配合适的GPU

* 添加GPU内存不足的处理

### 3. 修复结果提取逻辑

* 根据实际输出格式调整结果提取

* 支持多种输出格式

* 添加结果提取的容错处理

### 4. 添加命令行输出

* 保留日志文件记录

* 同时在命令行显示实时输出

* 添加进度显示

### 5. 优化脚本结构

* 简化网络配置

* 统一参数处理

* 添加错误处理

## 修复步骤

1. 修改脚本的通道选择逻辑
2. 修复GPU设备配置
3. 调整结果提取逻辑
4. 添加命令行输出
5. 优化脚本结构
6. 测试修复后的脚本

## 预期结果

1. 所有网络都能正确使用指定的通道配置
2. GPU设备配置正确，没有CUDA错误
3. 结果提取正确，能获取到acc和macro-f1指标
4. 命令行能看到实时训练输出
5. 脚本运行稳定，能完成所有实验
